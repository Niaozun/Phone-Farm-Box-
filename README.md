# 设备通信控制软件

## 项目简介

这是一个基于 Electron 和 Web 技术开发的设备通信控制软件，支持串口和网络通信，可以监控和控制多个通道的设备状态。

## 主要功能

- **多通道控制**：支持20个通道的状态监控和控制（HUB模式、OTG模式、关机）
- **多种通信方式**：支持串口通信和网络通信
- **实时监控**：监控电源电压、风扇电流、机箱温度等参数
- **数据可视化**：通过图表展示历史数据趋势
- **多主题支持**：支持深色和浅色主题切换
- **多语言支持**：支持多种语言界面

## 系统要求

- Windows 10/11 或 macOS 10.15+
- Node.js 20.x LTS
- npm 或 yarn
- 支持的串口设备（如CH340/CH341芯片的USB转串口设备）

## 快速开始

### 方法一：使用简易安装脚本（推荐）

1. 下载项目文件到本地
2. 找到并双击 `install-windows-simple.bat`（Windows）或 `install-macos.sh`（macOS）
3. 按照提示完成安装
4. 使用桌面快捷方式或运行 `run-app.bat`/`run-app.sh` 启动应用

### 方法二：手动安装

1. 确保已安装 Node.js 20.x LTS
2. 克隆或下载项目代码
3. 在项目目录中打开命令行
4. 安装依赖：
   ```bash
   npm install --legacy-peer-deps
   ```
5. 启动应用：
   ```bash
   npm start
   ```

## 使用说明

### 连接设备

1. **串口连接**：
   - 选择"串口连接"选项
   - 从下拉列表中选择可用的串口
   - 设置正确的波特率和其他参数
   - 点击"连接"按钮

2. **网络连接**：
   - 选择"网络连接"选项
   - 输入设备的IP地址和端口
   - 点击"连接"按钮

### 控制设备

- **总控制**：使用顶部的总控制按钮可以同时控制所有通道
- **单独控制**：点击每个通道卡片上的按钮可以单独控制该通道

### 监控数据

- 设备连接后，主界面会实时显示设备状态数据
- 点击"历史图表"可以查看历史数据趋势
- 可以选择不同的参数和时间范围查看

## 常见问题

### 1. 无法找到串口设备

- 确保设备驱动已正确安装
- 尝试重新插拔USB设备
- 检查设备管理器中是否有识别到串口设备
- 以管理员权限运行应用

### 2. 应用启动失败

- 确保Node.js版本为20.x LTS
- 尝试删除`node_modules`目录和`package-lock.json`文件，重新安装依赖
- 检查是否有错误信息输出

### 3. 连接设备后无法通信

- 确保波特率等参数设置正确
- 检查设备是否支持当前的通信协议
- 查看日志输出是否有错误信息

## 开发指南

### 项目结构

- `main.js` - Electron 主进程，负责窗口创建和通信
- `preload.js` - 预加载脚本，处理IPC通信
- `index.html` - 主界面
- `package.json` - 项目配置和依赖

### 技术栈

- Electron - 桌面应用框架
- Node.js SerialPort - 串口通信
- Tailwind CSS - UI样式
- Chart.js - 数据可视化
